Introduction

Introduction to Astar Network

Understand what Astar is, how it fits into the Polkadot ecosystem, and what makes it uniquely powerful for dApp developers.

Polkadot Parachain

Astar operates as a parachain on Polkadot, connecting to the relay chain via a parachain slot. It inherits Polkadot's pooled security without requiring its own validator set. Collators produce parachain blocks and submit them to the relay chain for finalization by Polkadot validators.

Dual VM: EVM via Frontier + WASM via ink!

Astar integrates Frontier — a Substrate-compatible EVM implementation — allowing unmodified Solidity/EVM bytecode to run natively. Alongside this, the pallet-contracts module enables WebAssembly smart contracts written in ink!, a Rust-based eDSL. Both VMs share the same address space and can interoperate through precompiles.

dApp Staking (Build2Earn)

Astar's signature feature: developers register their dApps on-chain. ASTR token holders stake on their favourite projects. At every block, staking rewards are distributed — up to 50% going to dApp developers. This creates a sustainable funding mechanism for open-source dApp developers independent of VC funding or token sales.

Agile Coretime (Polkadot 2.0)

With Polkadot 2.0, Astar participates in Agile Coretime — a more flexible resource allocation model replacing fixed parachain slot auctions. Coretime can be purchased on-demand or in bulk, enabling more cost-efficient block production and opening the door to more parachain teams joining the ecosystem.

Architecture

Astar Architecture

Built on Substrate with Polkadot's security model, Astar's runtime combines multiple pallets to deliver a feature-rich multichain environment.

⚙️

Substrate + Frontier EVM

The Astar runtime is built with FRAME pallets on Substrate. Frontier adds an Ethereum-compatible JSON-RPC layer and maps EVM accounts to Substrate accounts, allowing MetaMask and Hardhat to work without modification. Gas is denominated in ASTR.

📜

WASM via ink! Contracts

ink! is a Rust-based embedded domain-specific language for writing smart contracts compiled to WASM. Contracts are uploaded as WASM blobs and executed in a sandboxed environment with resource metering. ink! contracts can call EVM precompiles and vice versa.

🛡️

NPoS Consensus

Astar uses Polkadot's Nominated Proof-of-Stake for block finalization. Collators produce candidate blocks; Polkadot's validators finalize them. This means Astar has economic finality backed by DOT staking — significantly stronger security than standalone PoS chains.

⚡

Asynchronous Backing

Polkadot's Asynchronous Backing upgrade enables ~6s block production (down from ~12s) by pipelining block authoring and relay chain inclusion. Astar benefits from higher throughput and lower latency without changes to the application layer.

dApp Staking (Build2Earn)

Astar's flagship economic innovation — a protocol-level funding mechanism that turns staking into developer income.

How It Works

  1. Developer Registers dApp

    Any developer can register a deployed smart contract address (EVM or WASM) on-chain via the dApp Staking pallet. Registration requires a small deposit and community approval.

  2. Stakers Nominate

    ASTR token holders stake their tokens on one or more registered dApps. Staked tokens remain liquid (no unbonding cliff) and earn staking APY for the staker while simultaneously supporting the dApp.

  3. Rewards Distributed

    At each block, a portion of block rewards is split: ~50% goes to dApp developers (proportional to total stake on their dApp), and the remainder goes to stakers as staking APY. Rewards are claimable on-demand.

Reward Breakdown

Block rewards are allocated across three buckets:

  • dApp Developers: up to 50% of block rewards, split by staked weight
  • Stakers: base staking APY, encouraging participation
  • Collators: small collator reward for block production

Exact percentages are governed by on-chain parameters and can be adjusted via governance.

dApp Staking v3

dApp Staking v3 introduces tiered reward structures, hybrid staking periods, and improved governance controls. Developers can track their tier status and estimated APY in real-time via the Astar Portal dashboard at portal.astar.network.

ethers.js Example

import { ethers } from "ethers";

const provider = new ethers.JsonRpcProvider(
  "https://evm.astar.network"
);

// Check chain ID (should be 592n)
const network = await provider.getNetwork();
console.log("Chain ID:", network.chainId);

// Get latest block
const block = await provider.getBlock("latest");
console.log("Block:", block.number);

@polkadot/api Example

import { ApiPromise, WsProvider } from "@polkadot/api";

async function connect() {
  const provider = new WsProvider(
    "wss://rpc.astar.network"
  );
  const api = await ApiPromise.create({ provider });

  // Get runtime version
  const version = api.runtimeVersion;
  console.log("Runtime:", version.specName.toString());

  // Query dApp staking info
  const era = await api.query.dappStaking.currentEra();
  console.log("Current era:", era.toString());

  return api;
}

connect();
Substrate ↔ EVM Accounts

Astar uses a unified account model. An Ethereum address (H160) maps deterministically to a Substrate address (SS58) and vice versa. The unified-accounts pallet enables this mapping, so the same private key controls both account types.
