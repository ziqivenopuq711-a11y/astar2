Overview

What is Sonic L1?

Sonic is a next-generation EVM-compatible Layer-1 blockchain engineered for applications that demand extreme throughput, real-time finality, and low transaction costs.

Sonic L1 is the evolution of the Fantom Opera network — a ground-up redesign that preserves complete Ethereum compatibility while shattering the performance limits of traditional EVM chains. Where most EVM chains process a few thousand transactions per second at best, Sonic delivers up to 400,000 TPS with deterministic finality in under one second. Where Ethereum gas fees can spike to tens of dollars under load, Sonic maintains near-zero fees even at full capacity.

This leap in performance is not a compromise. Sonic is 100% backward-compatible with the Ethereum Virtual Machine. Every Solidity contract, every Hardhat and Foundry workflow, every Ethers.js and Viem integration, every wallet and block explorer — all work on Sonic without modification. Developers who know Ethereum can build on Sonic immediately, with their existing skills and tools, and immediately access the platform's performance advantages.

🧬
Evolved from Fantom
Sonic preserves the complete account and state history of Fantom Opera. Projects that ran on Fantom can migrate their contracts, user accounts, and token balances to Sonic without data loss, inheriting the platform's higher performance immediately.
🔗
Fully EVM Compatible
Deploy any Solidity or Vyper smart contract to Sonic without changes. Use MetaMask, WalletConnect, Hardhat, Foundry, Ethers.js, Viem, or any other Ethereum tooling. Sonic's JSON-RPC API is fully Ethereum-specification-compliant.
Core Technology

Built for Performance

Sonic's performance comes from four interconnected innovations: a faster VM, a more efficient state database, a next-generation consensus protocol, and a developer-aligned fee model.

⚙️
SonicVM
A redesigned EVM execution engine that compiles EVM bytecode into "super instructions" — composite opcodes that reduce interpreter dispatch overhead by up to 65%. SonicVM is 100% compatible with standard EVM bytecode and produces identical execution results at dramatically higher speed.
🗄️
SonicDB
A purpose-built state storage layer that separates live state from historical archive data. Live state is kept in a compact, cache-optimized structure for fast SLOAD/SSTORE operations. Historical data is compressed and archived separately, dramatically reducing node storage requirements.
🤝
Lachesis aBFT
An asynchronous Byzantine Fault Tolerant consensus protocol that achieves deterministic finality in under one second without requiring validators to vote in synchronised rounds. Lachesis can tolerate up to one-third of validators being malicious or offline while maintaining liveness and safety.
💰
FeeM (Fee Monetization)
A protocol-level mechanism that rebates up to 90% of the gas fees generated by a smart contract back to the contract's developer. Registered developers earn passive income from every transaction their users make — aligning developer incentives with network usage and growth.

Quick Start

Start Building on Sonic

From zero to deployed smart contract in minutes — using the same tools you already know from Ethereum.

  1. Add Sonic to your wallet

    Open MetaMask → Add network → Add manually. Enter: Network Name Sonic Mainnet, RPC URL https://rpc.soniclabs.com, Chain ID 146, Symbol S. For development, add Blaze testnet on Chain ID 64165 first.

  2. Get test tokens from the faucet

    Visit faucet.soniclabs.com, connect your wallet, and request free test S tokens for the Blaze testnet. Tokens arrive within seconds and are sufficient for extensive contract testing.

  3. Configure Hardhat or Foundry

    Add Sonic networks to your project config. In Hardhat, add sonicBlaze and sonic networks pointing to the respective RPC URLs. In Foundry, create a [rpc_endpoints] entry in foundry.toml. No other changes are needed — all your existing tasks and scripts work as-is.

  4. Deploy and verify

    Run forge script --rpc-url sonicBlaze --broadcast --verify or npx hardhat deploy --network sonicBlaze. Verify your contract on testnet.sonicscan.org and explore the transaction explorer to inspect your deployment.

  5. Register for FeeM

    Once deployed on mainnet, register your contract for Fee Monetization to earn up to 90% of gas fees your users pay. Registration is fully on-chain and permissionless. See the FeeM documentation for details.
