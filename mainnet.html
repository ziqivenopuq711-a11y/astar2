Overview

Mainnet Overview

Astar Mainnet is secured by Polkadot's NPoS relay chain and has been running in production since early 2022.

Polkadot Parachain

Astar Mainnet operates as a parachain on Polkadot, connected to the relay chain via a parachain slot. Security is pooled from Polkadot's validator set of hundreds of NPoS-elected validators. Collators (block producers) do not need to hold a security bond — the relay chain handles finalization.

Production Since Jan 2022

Astar Mainnet launched on Polkadot in January 2022, winning a parachain auction slot. It has processed millions of transactions and hosts a growing DeFi, NFT, and gaming ecosystem. All protocol upgrades are deployed to Shiden (canary network) before Astar Mainnet.

Dual Virtual Machines

Both execution environments are production-ready on mainnet:

  • EVM: Full Ethereum compatibility via Frontier. Chain ID 592. Use any Ethereum wallet, library, or toolchain.
  • WASM: ink! smart contracts via pallet-contracts. Deploy Rust-based dApps with full Substrate integration.
ASTR Token Economics

ASTR is the native utility token of Astar Mainnet. It is used for transaction fees (gas), dApp Staking (both staking and developer rewards), on-chain governance voting, and XCM fee payments. ASTR has 18 decimal places, matching the Ethereum standard.

Deployment

Deploying to Mainnet

Follow this production deployment checklist before going live on Astar Mainnet.

  1. Test Thoroughly on Shibuya

    Deploy and test your contract on Shibuya Testnet (Chain ID 81) first. Shibuya mirrors mainnet configuration — same pallet versions, same EVM behaviour. Run your full test suite and manual tests before considering mainnet deployment.

  2. Conduct a Security Audit

    For production smart contracts, engage a reputable auditor. Astar Foundation can recommend ecosystem partners. Publish your audit report publicly to build user trust and meet dApp Staking registration requirements.

  3. Configure Mainnet RPC

    Update your deployment scripts to use mainnet: url: "https://evm.astar.network", chainId: 592. Ensure your private key is secure — use a hardware wallet or secrets manager, never plain-text .env files in production.

  4. Deploy and Verify

    Run npx hardhat run scripts/deploy.js --network astar. Verify your contract on Blockscout immediately after deployment using the Hardhat verification plugin or manually via the Blockscout UI.

  5. Register for dApp Staking

    Navigate to portal.astar.network → dApp Staking → Register dApp. Provide your contract address, project metadata, and website. Community stakers can immediately begin nominating ASTR to your dApp to earn you block rewards.

Ecosystem

Astar Ecosystem

A growing ecosystem of DeFi protocols, DEXs, NFT platforms, and GameFi dApps — all benefiting from dApp Staking rewards.

💱

DeFi Protocols

Lending protocols, yield aggregators, and liquidity pools leverage Astar's EVM compatibility with XCM-bridged assets from across the Polkadot ecosystem including DOT, USDT, and bridged ETH.

🔄

DEXs

Decentralized exchanges on Astar benefit from Uniswap-compatible AMM designs, deep liquidity from native ASTR pairs, and XCM-sourced cross-chain token availability.

🖼️

NFTs

NFT platforms and marketplaces deploy ERC-721 and ERC-1155 contracts on Astar. The dApp Staking mechanism gives NFT projects a sustainable income stream beyond primary sales.

Shiden: Canary Network (Chain ID 336, SDN)

Shiden Network is Astar's canary network on Kusama — the experimental, production-grade sibling network. Chain ID 336, native token SDN. All Astar protocol upgrades (new pallets, EVM versions, dApp Staking v3) are deployed to Shiden weeks before Astar Mainnet. Shiden has its own DeFi ecosystem, dApp Staking registry, and Kusama parachain slot. RPC: https://evm.shiden.astar.network · Substrate: wss://rpc.shiden.astar.network.
