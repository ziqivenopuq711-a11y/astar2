Overview

Shibuya Overview

Shibuya is designed to be an accurate pre-production environment. What works on Shibuya will work on Astar Mainnet.

Purpose

Shibuya exists to let developers test smart contracts, dApp integrations, and XCM interactions in a risk-free environment with no real economic value. Tokens (SBY) are freely available and have no monetary value. Reset events may occur to keep the chain lightweight for testing purposes.

Protocol Parity

Shibuya tracks Astar Mainnet's runtime version closely — typically within one release. It runs the same FRAME pallets, the same Frontier EVM version, the same dApp Staking v3 implementation, and the same XCM configuration. Developers can be confident that behaviour observed on Shibuya reflects what they will see on mainnet.

Testing Pipeline

Recommended deployment progression for production readiness:

  1. Shibuya (Chain ID 81) — develop and iterate, free tokens, fast iteration
  2. Shiden (Chain ID 336) — canary network on Kusama, real economic value (SDN), test in production-like conditions
  3. Astar Mainnet (Chain ID 592) — production deployment on Polkadot with ASTR

MetaMask Setup Steps

  1. Open MetaMask network settings

    Click the network selector dropdown at the top of MetaMask. Choose "Add network" then "Add a network manually".

  2. Enter Shibuya parameters

    Network Name: Shibuya Testnet, RPC URL: https://evm.shibuya.astar.network, Chain ID: 81, Symbol: SBY.

  3. Save and switch

    Click Save. MetaMask switches to Shibuya and your SBY balance (initially 0) appears. Proceed to the faucet.

  4. Get SBY from the faucet

    Visit portal.astar.network, connect your wallet on Shibuya, and request free SBY tokens. Tokens are sent within seconds.

Testing

Testing & Debugging

Best practices for testing smart contracts against Shibuya using Hardhat, gas estimation, and debugging techniques.

Hardhat Test Example

// test/MyContract.test.js
const { expect } = require("chai");
const { ethers } = require("hardhat");

describe("MyContract on Shibuya", function () {
  let contract;
  let owner;

  before(async function () {
    // Uses shibuya network from hardhat.config.js
    [owner] = await ethers.getSigners();
    const Factory = await ethers.getContractFactory("MyContract");
    contract = await Factory.deploy();
    await contract.waitForDeployment();
  });

  it("should return correct value", async function () {
    const val = await contract.getValue();
    expect(val).to.equal(42n);
  });

  it("should emit event on setValue", async function () {
    await expect(contract.setValue(100n))
      .to.emit(contract, "ValueSet")
      .withArgs(100n);
  });
});

// Run against Shibuya:
// npx hardhat test --network shibuya

Gas Estimation

// Estimate gas before deploying
const Factory = await ethers.getContractFactory("MyContract");
const deployTx = await Factory.getDeployTransaction();

const gasEstimate = await provider.estimateGas(deployTx);
const gasPrice = await provider.getFeeData();

const estimatedCostWei = gasEstimate * gasPrice.gasPrice;
const estimatedCostSBY = ethers.formatEther(estimatedCostWei);
console.log(`Deploy cost: ~${estimatedCostSBY} SBY`);
Debugging Tips
  • Use console.log in Hardhat tests with hardhat-toolbox — logs appear in test output even for Shibuya forks
  • Check transaction receipts for status: 0 (revert) and inspect revertReason via eth_call replay
  • Use Blockscout at blockscout.com/shibuya to decode revert messages visually
  • For Substrate-level issues (dApp Staking, XCM), use shibuya.subscan.io to inspect extrinsic events
