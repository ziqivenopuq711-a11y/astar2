Sonic Explorer Overview

A comprehensive tool for monitoring, auditing, and analyzing all on-chain activity across the Sonic L1 network.

The Sonic block explorer is the primary transparency interface for the Sonic L1 network. It provides real-time visibility into every aspect of on-chain activity: individual transactions and their execution status, block production and validator information, account balances and transaction histories, deployed smart contracts and their interactions, and live network health metrics.

Because Sonic achieves sub-second finality, the explorer updates almost instantly after each new block is produced. Unlike explorers on networks with 12-second or longer block times, the Sonic explorer reflects the current state of the network with minimal lag — making it an indispensable tool for monitoring live deployments, verifying transaction outcomes in real time, and diagnosing on-chain issues as they occur rather than minutes later.

🧱
Block Browser
Browse every block produced on Sonic. View the block height, timestamp, validator who produced it, number of transactions included, gas used vs. limit, block hash, and parent hash for full chain verification.
💸
Transaction Lookup
Search any transaction by hash, account address, block number, or contract address. View full execution details including method called, arguments decoded, event logs emitted, and exact gas usage and cost.

Real-time charts displaying TPS, gas price trends, active address counts, token transfer volumes, and validator performance metrics. Essential for understanding network load and planning optimal transaction timing.

The Sonic explorer also supports token tracking for all ERC-20, ERC-721, and ERC-1155 tokens deployed on the network. Users can view token supply, holder distribution, transfer histories, and contract metadata — all presented in a clean interface that makes Sonic's transparent on-chain activity accessible to both technical developers and non-technical end users.

Transactions

Transaction Tracking

Search, verify, and analyze any transaction on Sonic — from simple S token transfers to complex multi-step DeFi interactions.

Every transaction submitted to Sonic receives a unique transaction hash (txHash) — a 32-byte hexadecimal identifier that can be used to look up full execution details in the block explorer. Because Sonic achieves deterministic finality in under one second, a transaction that appears in the explorer as "Success" is guaranteed to be final; there is no need to wait for additional block confirmations as there is on proof-of-work or probabilistic finality networks.

The transaction detail view in the Sonic explorer provides a wealth of information for developers and users: the sender address (from), recipient or contract address (to), value of S tokens transferred, the exact input data sent to the contract (decoded into human-readable function calls when the ABI is available), the list of events emitted during execution (decoded into their named parameters), gas limit set by the sender, actual gas used, effective gas price, and the resulting fee paid in S tokens.

Understanding Transaction Status

Status Meaning What to do
Success Transaction executed and finalized on-chain No action needed — state changes are permanent
Pending Transaction submitted but not yet included in a block Wait for inclusion; on Sonic this typically takes <1 second
Failed Transaction reverted during execution Check the revert reason — insufficient gas, failed require(), or contract logic error

Gas costs on Sonic are significantly lower than on Ethereum mainnet due to the network's high throughput and efficient fee market. The near-zero fees make micro-transactions economically viable — a transaction that would cost $5–$50 on Ethereum typically costs fractions of a cent on Sonic, enabling use cases that are simply impractical on higher-fee networks.

Searching Transactions

The explorer's universal search bar accepts: transaction hashes (0x-prefixed 64-character hex strings), wallet addresses (0x-prefixed 40-character hex strings), ENS names (where supported), block numbers (plain integers), and token names or symbols. Results are returned instantly from the indexed database, with no delay even on historical queries spanning millions of blocks.

Smart Contracts

Smart Contract Verification

Verify your contracts on-chain to enable source code transparency, ABI decoding, and interactive read/write interfaces.

Smart contract verification is the process of matching a deployed contract's on-chain bytecode with human-readable Solidity (or Vyper) source code. Once verified, the Sonic explorer can display the contract's source code, ABI (Application Binary Interface), and provide an interactive read/write panel that allows anyone to call contract functions directly from their browser without needing custom tooling.

Verification on Sonic works identically to Etherscan verification on Ethereum — because Sonic is fully EVM-compatible, the same compilation toolchains (solc, vyper) produce the same deterministic bytecode. The verification process involves submitting the source code, compiler version, optimization settings, and constructor arguments to the explorer's API, which recompiles the code and checks that the resulting bytecode matches what is deployed on-chain.

  1. Navigate to the contract address

    Search for your deployed contract address in the Sonic explorer and open the "Contract" tab on the detail page.

  2. Click "Verify and Publish"

    Select the compilation method: single file, standard JSON input, or multi-file. Choose the Solidity compiler version matching what you used for deployment.

  3. Submit source code

    Paste your source code or upload the standard JSON input file. Include constructor arguments in ABI-encoded format if your contract's constructor accepts parameters.

  4. Use the read/write panel

    Once verified, the "Read Contract" and "Write Contract" tabs become available. Connect MetaMask to call state-changing functions directly through the explorer UI.

Hardhat and Foundry both support automated verification on Sonic-compatible explorers via their built-in verification plugins. Configure the explorer API URL in your deployment script to submit verification automatically after each deployment, eliminating the need for manual steps.

The Sonic explorer's statistics dashboard aggregates data across all validator nodes to produce a real-time view of network health. Key metrics include the current transaction per second rate (sampled over 1-minute, 5-minute, and 60-minute windows), the mempool depth (number of pending transactions awaiting inclusion), current minimum and average gas prices, validator uptime percentages, and epoch-by-epoch staking reward distributions.

Total Value Locked (TVL) data is tracked per protocol and per asset class. Users can view which DeFi applications are accumulating the most liquidity, observe liquidity flow trends, and understand which segments of the Sonic DeFi ecosystem are growing fastest. This data is particularly valuable for FeeM-registered developers, who can correlate their TVL growth with their earned fee rebates and optimize their applications for maximum revenue.

Network security metrics are also prominently displayed: the total amount of S tokens staked across all validators, the staking ratio (staked S / total S supply), the current number of active validators, and the geographic distribution of validator nodes. These metrics help the community assess the decentralization and censorship resistance of the Sonic network at a glance.

📡 API Access

All explorer data is available via REST API. Query blocks, transactions, accounts, tokens, and statistics programmatically by appending /api to the explorer base URL. Full API documentation is available on the explorer website. Rate limits apply for unauthenticated requests; register for an API key for higher throughput.
