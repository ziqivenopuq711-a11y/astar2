Libraries

Sonic SDK & Libraries

All major Ethereum libraries are fully compatible with Sonic. No custom forks, no special adapters — just point to the Sonic RPC and start building.

Sonic's full EVM compatibility is not limited to smart contract execution — it extends to every layer of the developer stack. The JSON-RPC API exposed by Sonic nodes is fully compliant with the Ethereum JSON-RPC specification, which means every library, SDK, and tool that speaks Ethereum JSON-RPC works with Sonic identically. You don't need a Sonic-specific SDK because the entire Ethereum ecosystem is already your Sonic SDK.

🔷
ethers.js v6
The most widely-used Ethereum JavaScript library. Connect to Sonic by passing the Sonic RPC URL to JsonRpcProvider. All ethers.js features — contract deployment, event listening, ENS resolution, multicall — work without modification on Chain ID 146.
import { ethers } from "ethers";
const p = new ethers.JsonRpcProvider(
  "https://rpc.soniclabs.com"
);
🌐
web3.js v4
The original Ethereum JavaScript library remains fully supported on Sonic. Create a Web3 instance with the Sonic HTTP or WebSocket endpoint to access all standard eth, net, and personal modules with sub-second response times.
import Web3 from "web3";
const web3 = new Web3(
  "https://rpc.soniclabs.com"
);
⚡
viem
The modern, type-safe Ethereum library with an excellent developer experience. Define a Sonic chain configuration and use viem's public client to interact with the network. Full TypeScript types, tree-shaking, and minimal bundle size.
import { createPublicClient, http }
  from "viem";
const client = createPublicClient({
  transport: http("https://rpc.soniclabs.com")
});
🐍
web3.py
Python developers can use web3.py to interact with Sonic. The library's HTTPProvider and WebsocketProvider connect to Sonic endpoints seamlessly. Popular in backend services, data pipelines, and bot development for DeFi protocols.
from web3 import Web3
w3 = Web3(Web3.HTTPProvider(
  "https://rpc.soniclabs.com"
))

Beyond these core libraries, Sonic is compatible with the full spectrum of Ethereum ecosystem tooling: OpenZeppelin contract libraries (ERC-20, ERC-721, ERC-1155, access control, upgradeable proxies), Uniswap v2/v3 factory deployments, Chainlink oracle integrations, Safe multisig deployments, and The Graph subgraph indexing. Any protocol or infrastructure that supports EVM-compatible chains can be configured to use Sonic with a simple RPC endpoint change.

Tooling

Smart Contract Tooling

Configure your preferred development framework to target Sonic mainnet and Blaze testnet with just a few lines of config.

Hardhat Configuration

Add Sonic networks to your hardhat.config.js file. Sonic supports all standard Hardhat features including compilation, testing with Hardhat Network forking, deployment scripts, and the hardhat-etherscan verification plugin.

// hardhat.config.js
require("@nomicfoundation/hardhat-toolbox");

module.exports = {
  solidity: "0.8.24",
  networks: {
    sonic: {
      url: "https://rpc.soniclabs.com",
      chainId: 146,
      accounts: [process.env.PRIVATE_KEY],
    },
    sonicBlaze: {
      url: "https://rpc.blaze.soniclabs.com",
      chainId: 64165,
      accounts: [process.env.PRIVATE_KEY],
    },
  },
  etherscan: {
    apiKey: {
      sonic: process.env.SONICSCAN_API_KEY,
    },
    customChains: [{
      network: "sonic",
      chainId: 146,
      urls: {
        apiURL: "https://api.sonicscan.org/api",
        browserURL: "https://sonicscan.org",
      },
    }],
  },
};

Foundry Configuration

Foundry's foundry.toml can be configured with Sonic RPC endpoints for both deployment and testing. Use forge script with the --rpc-url sonic flag after configuring the alias below.

# foundry.toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]
solc = "0.8.24"

[rpc_endpoints]
sonic = "https://rpc.soniclabs.com"
sonic_blaze = "https://rpc.blaze.soniclabs.com"

[etherscan]
sonic = { key = "${SONICSCAN_API_KEY}", url = "https://api.sonicscan.org/api" }

# Deploy command:
# forge script script/Deploy.s.sol --rpc-url sonic --broadcast --verify

Remix IDE

Remix IDE works with Sonic directly through the "Injected Provider" option (MetaMask connected to Sonic mainnet or Blaze testnet) or by using the "External HTTP Provider" option with the Sonic RPC URL. The Remix Sonic plugin (available in the plugin manager) provides one-click deployment and verification for developers who prefer a browser-based IDE without local toolchain setup.

Supported JSON-RPC Methods

MethodDescription
eth_blockNumberReturns the current latest block number
eth_getBalanceReturns the S token balance of a given address
eth_sendRawTransactionSubmits a signed transaction to the network
eth_callExecutes a read-only contract call without creating a transaction
eth_estimateGasEstimates the gas cost of a transaction
eth_getTransactionReceiptReturns the receipt for a given transaction hash
eth_getLogsReturns logs matching a filter (address, topics, block range)
eth_subscribeWebSocket: subscribe to new blocks, pending transactions, or log events
eth_getCodeReturns the bytecode of a deployed contract
eth_chainIdReturns Chain ID 0x92 (146 decimal) for mainnet

WebSocket Subscriptions

// Subscribe to new block headers via WebSocket
const ws = new WebSocket("wss://rpc.soniclabs.com/ws");

ws.onopen = () => {
  ws.send(JSON.stringify({
    jsonrpc: "2.0",
    id: 1,
    method: "eth_subscribe",
    params: ["newHeads"]
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.method === "eth_subscription") {
    console.log("New block:", data.params.result.number);
  }
};

WebSocket connections to Sonic's RPC nodes provide real-time event streaming. Because Sonic produces blocks in under one second, WebSocket subscriptions for newHeads deliver block events at a pace that would be impractical to poll via HTTP. Use WebSocket subscriptions for live dashboards, trading bots, and any application that requires immediate reaction to on-chain events.
