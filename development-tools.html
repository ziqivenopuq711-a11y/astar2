Build on Astar with Familiar Tools

Astar's EVM is fully compatible with the Ethereum toolchain. Use ethers.js, viem, Hardhat, or Foundry exactly as you would on Ethereum — just point your RPC at Astar. For Substrate-native development, use @polkadot/api and Subxt for full runtime access.

Libraries

JavaScript / TypeScript Libraries

Connect to Astar's EVM with Ethereum-compatible libraries, or use Polkadot.js for full Substrate runtime access.

EVM Libraries

ethers.js v6
import { ethers } from "ethers";

// Connect to Astar Mainnet
const provider = new ethers.JsonRpcProvider(
  "https://evm.astar.network"
);

// Connect to Shibuya Testnet
const testProvider = new ethers.JsonRpcProvider(
  "https://evm.shibuya.astar.network"
);

// Deploy a contract
const signer = new ethers.Wallet(PRIVATE_KEY, provider);
const factory = new ethers.ContractFactory(abi, bytecode, signer);
const contract = await factory.deploy();
await contract.waitForDeployment();
console.log("Deployed:", await contract.getAddress());
viem
import { createPublicClient, http } from "viem";
import { defineChain } from "viem/chains";

const astar = defineChain({
  id: 592,
  name: "Astar",
  nativeCurrency: { name: "Astar", symbol: "ASTR", decimals: 18 },
  rpcUrls: {
    default: { http: ["https://evm.astar.network"] },
  },
  blockExplorers: {
    default: { name: "Subscan", url: "https://astar.subscan.io" },
  },
});

const client = createPublicClient({ chain: astar, transport: http() });
const blockNumber = await client.getBlockNumber();
console.log("Block:", blockNumber);

Substrate Libraries

@polkadot/api
import { ApiPromise, WsProvider } from "@polkadot/api";

const api = await ApiPromise.create({
  provider: new WsProvider("wss://rpc.astar.network"),
});

// Query dApp Staking era
const era = await api.query.dappStaking.currentEra();

// Query account balance
const { data: balance } = await api.query.system.account(ADDR);
console.log("Free:", balance.free.toHuman());

// Listen to new blocks
api.rpc.chain.subscribeNewHeads((header) => {
  console.log("Block #", header.number.toNumber());
});
Subxt (Rust)
# Cargo.toml
[dependencies]
subxt = "0.35"

# Generate type-safe bindings
subxt metadata --url wss://rpc.astar.network \
  > astar_metadata.scale
Frameworks

Smart Contract Tooling

Configure Hardhat or Foundry for EVM contracts, or use cargo-contract for ink! WASM contracts.

Hardhat Configuration

// hardhat.config.js
require("@nomicfoundation/hardhat-toolbox");

module.exports = {
  solidity: "0.8.24",
  networks: {
    astar: {
      url: "https://evm.astar.network",
      chainId: 592,
      accounts: [process.env.PRIVATE_KEY],
    },
    shibuya: {
      url: "https://evm.shibuya.astar.network",
      chainId: 81,
      accounts: [process.env.PRIVATE_KEY],
    },
    shiden: {
      url: "https://evm.shiden.astar.network",
      chainId: 336,
      accounts: [process.env.PRIVATE_KEY],
    },
  },
  etherscan: {
    apiKey: {
      astar: "YOUR_BLOCKSCOUT_API_KEY",
    },
    customChains: [{
      network: "astar",
      chainId: 592,
      urls: {
        apiURL: "https://blockscout.com/astar/api",
        browserURL: "https://blockscout.com/astar",
      },
    }],
  },
};

Foundry Configuration

# foundry.toml
[profile.default]
src = "src"
out = "out"
libs = ["lib"]

[rpc_endpoints]
astar = "https://evm.astar.network"
shibuya = "https://evm.shibuya.astar.network"
shiden = "https://evm.shiden.astar.network"

# Deploy to Astar Mainnet
forge script script/Deploy.s.sol \
  --rpc-url astar \
  --broadcast \
  --verify \
  --verifier blockscout \
  --verifier-url https://blockscout.com/astar/api

# Deploy to Shibuya Testnet
forge script script/Deploy.s.sol \
  --rpc-url shibuya \
  --broadcast

ink! Contracts (cargo-contract)

# Install cargo-contract
cargo install cargo-contract

# Create a new ink! project
cargo contract new my_dapp
cd my_dapp

# Build WASM contract
cargo contract build --release

# Deploy to Shibuya
cargo contract instantiate \
  --constructor new \
  --args 42 \
  --suri //Alice \
  --url wss://rpc.shibuya.astar.network

Supported EVM Methods

All standard Ethereum JSON-RPC methods are supported: eth_getBalance, eth_sendRawTransaction, eth_call, eth_getLogs, eth_getTransactionReceipt, net_version, eth_chainId, eth_blockNumber, and all filter/subscription methods via WebSocket (eth_subscribe).
